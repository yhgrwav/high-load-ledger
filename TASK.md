Техническое Задание: High-Load Distributed Ledger Core

## 🎯 Цель проекта
Разработать отказоустойчивую систему для обработки финансовых транзакций, способную выдерживать высокие нагрузки за счет асинхронной модели обработки данных.

## 🛠 Технологический стек
- **Язык:** Go 1.25+
- **Взаимодействие:** gRPC (Protobuf)
- **Брокер сообщений:** Apache Kafka
- **Кэширование и Идемпотентность:** Redis
- **Хранилище данных:** PostgreSQL
- **Инфраструктура:** Docker, Docker Compose, Makefile

## 🏗 Архитектура
Проект реализуется в виде двух микросервисов:

### 1. API Gateway (Producer)
- Принимает gRPC-запросы на перевод денег.
- Проверяет `idempotency_key` в Redis (защита от дублей).
- Реализует Rate Limiting (защита от перегрузки).
- Отправляет валидные транзакции в Kafka (топик `tx.pending`).
- Возвращает мгновенный ответ `ACCEPTED`.

### 2. Ledger Worker (Consumer)
- Читает транзакции из Kafka пачками (Batching).
- Проверяет баланс пользователя в БД.
- Выполняет перевод в рамках ACID транзакции (Postgres).
- Реализует Graceful Shutdown для предотвращения потери данных при остановке.

## 📈 Критерии High-Load
- Использование **Batch Insert** для записи истории транзакций.
- Минимизация блокировок в БД (правильные уровни изоляции).
- Масштабируемость: возможность запустить несколько воркеров на одну очередь Kafka.
